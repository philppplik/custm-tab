Getting Started

  1. Open Settings Click the cust*m Tab icon in your Chrome toolbar, or go to chrome://extensions → Details → Extension options.
  2. Enter your custom URL Type any web address (e.g. https://notion.so, https://chat.openai.com) or a local file path (e.g. file:///C:/Users/you/dashboard.html).
  3. Click Save Open a new tab — your custom page loads instantly. That's it.

Using Local Files

You can set any local file as your new tab page. Chrome requires a one-time permission for this.

Step 1: Enable file access

  1. Go to chrome://extensions
  2. Find cust*m Tab → click Details
  3. Enable "Allow access to file URLs"

Step 2: Set the file path

Enter the full file path in the URL field, e.g.:

file:///C:/Users/you/Documents/my-page.html

⚠

Chrome restricts file:// access for security. This is a one-time setup — you only need to do this once.

Hide URL vs Redirect Mode

The "Hide target URL in address bar" toggle controls how your custom page is loaded.

Hide URL ON (iframe)Hide URL OFF (redirect)

💡

Leave Hide URL ON by default. If a page shows "Page blocks embedding", switch it OFF for that specific site.

Troubleshooting

"Page blocks embedding" error ›

The target site sets X-Frame-Options: deny to prevent being embedded in iframes. This is a safety feature — you cannot override it.

Fix: In Settings, turn OFF "Hide target URL in address bar" and Save. The page will open directly instead of in an iframe.

Local file doesn't load / blank page ›

Chrome requires "Allow access to file URLs" permission for extensions to load local files.

Fix: Go to chrome://extensions → Details → Enable "Allow access to file URLs".

"Extension disabled" popup ›

Chrome may disable extensions after updates or sync changes. This is a known Chrome behavior.

Fix: Go to chrome://extensions and re-enable the extension. Your settings are preserved.

Page loads slowly ›

JS-heavy pages take time to render. Enable "Preload page every 10 minutes" to keep the page warm in the background.

Note: This opens the target page in a hidden tab to keep it loaded.

Address bar shows a long URL after loading ›

This happens when "Hide target URL" is OFF. To hide it, enable the toggle in Settings — your new tab will show chrome-extension://… instead.
